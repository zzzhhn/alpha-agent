# Alpha-Agent v3 → v4 Gap Audit

**Audit date:** 2026-05-01 (v3 just shipped end-to-end)
**Method:** 4 parallel research agents dispatched via superpowers / everything-claude-code framework, each covering one orthogonal lane. Lane outputs cross-validated, deduped, ranked by impact × cost.
**Outcome:** 50 raw gaps → 5-tier prioritized punch list + Top-5 immediate-action set.

## 1. Cross-lane convergence themes

5 themes appeared in ≥2 lanes independently. Convergence = strongest signal that the gap is real, not lane-specific.

| # | Theme | Lanes | Why it's #1-tier |
|---|-------|-------|------------------|
| 1 | **Look-ahead bias** (fundamentals broadcast w/o publish-lag + no operand audit) | B#2/B#3 + C#1/C#3 | Any factor using `eps`, `revenue`, `fcf`, `ebitda`, etc. structurally inflated. |
| 2 | **Survivorship bias** (panel = today's SP100 backfilled 1y) | B#1 + C#2 | Adds 1-3% annualized phantom return for any survival-correlated factor. |
| 3 | **Evaluation rigor holes** (overlapping WF + no purge + no ICIR + no significance test) | D#1/D#2/D#3/D#4 | Selection-bias engine: rolling-stat factors get IC inflated 10-40% by leakage; Zoo amplifies via implicit beauty contest. |
| 4 | **Transaction-cost simplification** (flat bps; no slippage / borrow / impact) | A#9 + B#4/B#5 | Distorts long-short Sharpe by 0.2-0.5; high-turnover factors over-credited. |
| 5 | **No factor performance database** (Zoo lives in frontend localStorage only) | A#1/A#2/A#3 | Closed-loop research, regime-aware weighting, decay alerts, genetic search all blocked on this single piece of infra. |

Themes 1+2+3 = trustworthiness of v3 results. Theme 4 = decision realism. Theme 5 = platform evolution from "calculator" to "research agent".

## 2. Tier 1 — Trust blockers

Until these ship, any v3 backtest / screener number is a function of bias as much as alpha.

| Item | Severity | Cost | Source | Concrete landing |
|------|----------|------|--------|-------------------|
| **T1.1** Fundamentals +45d publish-lag | Critical | M | B#2, C#1 | `kernel.build_data_dict()` ~line 80-90: shift each fundamental array along time axis by `report_date - date` (or flat 45d if report_date unavailable) |
| **T1.2** Operand look-ahead tamper-test | Critical | S | B#3 | `tests/test_no_lookahead.py`: scramble `close[t+1:]`, run every op in `OPS`, assert `factor[t]` unchanged |
| **T1.3** Walk-forward purge + embargo | Critical | S | D#1 | `factor_backtest.py` WF loop: drop last `max(lookback, fwd_horizon)` of each train slice; skip first `embargo_days` of each test slice |
| **T1.4** ICIR + IC t-stat + p-value | Critical | S | D#2, D#3 | `_split_metrics`: add `ic_std`, `icir = ic_mean / ic_std × √252`, `ic_pvalue = 2 × (1 − cdf(\|t\|))` |
| **T1.5** Survivorship-corrected universe | Critical | M-L | B#1, C#2 | New `scripts/build_pit_universe.py`: monthly SP100 membership snapshots (Wikipedia revisions or iShares OEF holdings); `is_member[t,n]` mask consumed by backtest |

T1.1-T1.4 are S/M (~600 LOC total). T1.5 needs new data sourcing — separable.

## 8. Top-5 immediate-action list (strict ROI = Severity × 1/Cost)

1. **T1.2 operand look-ahead tamper-test** — ~80 LOC pytest file; prevents whole class of future regressions; zero runtime impact.
2. **T1.4 ICIR + IC p-value** — 4-line addition to `_split_metrics`; immediately flags "IC=0.02 ± 0.15" pseudo-factors that currently pass.
3. **T1.3 WF purge + embargo** — ~30-line modification to `factor_backtest.py` WF loop; combined with T1.4 catches ~80% of current overfit false positives (Lane D estimate).
4. **T1.1 fundamentals publish-lag** — `kernel.build_data_dict` +45d shift; first time fundamental factors produce trustworthy results. Same MCP pull also unlocks T3 earnings window mask.
5. **T2.1 deflated Sharpe + Zoo cross-correlation warning** — first honest answer to "how many independent alphas does the Zoo actually contain?"

5-item bundle: ~600 LOC, all S/M cost, deployable in 2-3 sessions.

After this bundle: **alpha-agent enters the regime where users can act on backtest numbers without being structurally misled.**

T1.5 (survivorship), T4.1 (Factor DB), T4.3 (genetic mining) = three remaining major engineering blocks corresponding to data foundation, memory foundation, autonomous-search foundation.

## 10. Methodology note

The 4 lanes were selected to be orthogonal:
- **A — Strategy/methodology**: vs SOTA quant agent papers (AlphaAgent, RD-Agent, AutoAlpha, QuantaAlpha) and frameworks (Qlib, WorldQuant 101).
- **B — Backtest realism + risk**: vs production backtesters (Qlib, Zipline, Lean).
- **C — Data infra**: vs typical quant data needs + cross-check installed `financial-datasets` MCP.
- **D — Evaluation rigor**: vs ML/quant validation best practice (López de Prado, deflated Sharpe, purged CV).

Lane B's quant-analyst agent and Lane C's general-purpose agent **independently surfaced the same fundamentals look-ahead bug** (B identified it via `kernel.build_data_dict` direct broadcasting; C identified it via missing `report_period` field). Cross-lane confirmation lifted this from "lane-specific suggestion" to Tier-1 trust blocker.

Lane D produced the most actionable synthesis: "the most likely reason a factor currently passes alpha-agent's bar" = rolling-lookback × overlapping-WF leakage × selection-from-many siblings, stacking geometrically. This pinned the abstract "evaluation rigor" gap onto a concrete failure mechanism.

The 4-lane parallel dispatch consumed ~250K tokens total (50K-75K per lane) and ~5 minutes wall-clock. Roughly 5× cheaper than serial deep audit and produced cross-validation that single-pass review wouldn't have.
